Recommendations
🔒
Upgrade Rate Limiting to Redis (Upstash)
The current in-memory rate limiter resets on every server restart and doesn't share state across multiple Vercel function instances. Upstash Redis (free tier: 10,000 req/day) provides persistent, distributed rate limiting with zero ops overhead.
High priority • ~2 hours • npm install @upstash/ratelimit @upstash/redis
📧
Add Email Delivery Status Tracking
Currently, if Resend fails mid-broadcast (network error, API limit), some students receive the email and others don't — with no visibility. Add a broadcast_logs table tracking event_id, recipient_email, status (sent/failed), and timestamp. Resend's webhooks can update this in real-time.
High priority • ~4 hours
📊
Add Error Monitoring (Sentry)
There is no error monitoring in production. Sentry's free tier captures unhandled exceptions, API errors, and performance traces. It would surface issues like failed DB queries, Resend timeouts, or slow scan response times before students report them.
Medium priority • ~1 hour • npm install @sentry/nextjs
📱
Add PWA Support for Scanner Pages
Facilitators use the scanner on mobile. Adding a PWA manifest and service worker lets them install PharmaTrack to their home screen, access it faster, and keep the camera active without browser chrome getting in the way. Critical for event-day reliability.
Medium priority • ~3 hours • next-pwa or manual manifest
🗄️
Add Database Indexes on Hot Columns
As attendance records grow (800 students × many events), queries filtering by student_id, event_id, status, and created_at will slow down. Add indexes: CREATE INDEX ON attendance_records(event_id), CREATE INDEX ON attendance_records(student_id), and a composite on (event_id, status).
Medium priority • 30 minutes • Run via Supabase SQL editor
🔔
Notify At-Risk Students Automatically
The reports page identifies students below 75% attendance but has no way to notify them. Add a weekly cron job (Vercel Cron or Supabase Edge Function) that emails flagged students a summary of their attendance and a warning. Resend handles the delivery.
Low priority • ~4 hours
🧪
Add Integration Tests for Critical Paths
The scan, register, and event-creation flows have no automated tests. A minimal Playwright or Vitest integration suite covering happy-path + edge cases (expired QR window, duplicate scan, unapproved student) would prevent regressions at exam time.
Low priority • ~1 day
